# CES PMR Indexing Design

CES PMR indexing is an active-design-only publication-visible doctrine. CES indexes PMR; CES does not replace PMR. CES summarizes event state and accelerates retrieval while PMR remains the evidentiary replay store. Source expansion and human review remain required before decisions.

This design does not change runtime behavior, emit runtime index artifacts, enable similarity search, perform cross-user matching, federate similarity, replace PMR source artifacts, delete replay lineage, write memory, admit Atlas memory, train models, skip review, export traces, federate PMR, release product, certify truth, authorize final answers, or grant accepted-evidence authority.
